# Express Subtitle Export API

This service wraps the existing Next.js subtitle export flow and returns the transparent subtitle video in a single API response.

## What It Does

There are two supported workflows:

1. `mediaId` workflow:
   - Reads subtitles by VOD ID and renders transparent subtitle video.
2. File upload workflow:
   - Accepts a video/audio file from Postman.
   - Uploads it to Phont backend.
   - Polls processing status every 10 seconds.
   - When status becomes `COMPLETED`, fetches generated subtitles.
   - Renders transparent subtitle video and returns it in the same response.

## Endpoints

- `GET /health`
- `POST /api/subtitle-overlay/export`
- `POST /api/subtitle-overlay/export/:mediaId`
- `POST /api/subtitle-overlay/export-upload`
- `POST /api/subtitle-overlay/export-upload/:source/:pluginVodId`

## Request Body

```json
{
  "mediaId": "699f8db8e9492e363392611c",
  "language": "en",
  "model": "gpt-4.1-mini",
  "duration": 49,
  "fps": 60,
  "format": "prores4444-mov",
  "maxExpression": 1.0,
  "threshold": 0.2,
  "isRTL": false
}
```

Notes:
- `mediaId` is required (either in body or route param).
- `fps` minimum is `60`.
- `format` can be `prores4444-mov` or `vp9-alpha-webm`.
- If `duration` is omitted, service derives it automatically.

## File Upload Request (Postman)

Use `multipart/form-data`:

- `file`: your video/mp3 file (required)
- `source`: optional, default `postman-upload`
- `pluginVodId`: optional, autogenerated if missing
- `name`: optional title for backend upload job
- `fps`: optional, default `60` (minimum `60`)
- `format`: optional (`prores4444-mov` or `vp9-alpha-webm`)
- `duration`: optional override duration in seconds
- `maxExpression`, `threshold`, `isRTL`: optional render controls

Example cURL:

```bash
curl -X POST "http://127.0.0.1:5055/api/subtitle-overlay/export-upload" \
  -H "Accept: */*" \
  -F "file=@/absolute/path/to/input.mp3" \
  -F "source=postman-upload" \
  -F "fps=60" \
  -F "format=prores4444-mov" \
  --output subtitles-overlay.mov
```

## Run

1. Ensure Next app is running (it must expose `/api/subtitle-export/*`).
2. Configure environment:
   - Copy `.env.example` to `.env` and fill values.
3. Start this service:

```bash
pnpm --filter @phont/subtitle-export-api dev
```

## cURL Example

```bash
curl -X POST "http://127.0.0.1:5055/api/subtitle-overlay/export/699f8db8e9492e363392611c" \
  -H "Content-Type: application/json" \
  --output subtitles-overlay.mov
```

## Important

- This service does not replace existing frontend export flow.
- It reuses the same Next.js renderer pipeline, so subtitle styling/animation stays identical.
- File upload flow depends on Phont ingestion/status APIs and requires a valid plugin API key.

## Auto-Login For 401

If source API returns `401 Authentication required`, set:

- `SUBTITLE_SOURCE_AUTO_LOGIN_USERNAME`
- `SUBTITLE_SOURCE_AUTO_LOGIN_PASSWORD`

On a `401`, the service will call `/login`, store `auth_token + session_id` in memory, and retry the request automatically.

You can verify auth status via:

- `GET /health` -> `sourceAuth` object
- `GET /health` -> `pluginFlow` object
